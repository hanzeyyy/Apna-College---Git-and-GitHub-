Git and GitHub
Git: Version Control System is a tool that helps us to track changes in a code.
Git is a version control system. It is:
1. Free and open source 
2. popular 
3. Fast and scalable				
It is used for 2 main purpose:
1. Collaboration
2. To track the history of our code/project
Github:
Website that allows developers to store and manage their code using Git.
md -> mark down 
cd -> change directory  
ls -> list files 
ls —a > all list files (including hidden files)
cd .. -> get out of a directory 
mkdir “directory name” -> makes new directory  
Configuring Git
git config —global user.name “My Username” git config —global user.email “myemail@gmail.com”
git config —list 
Clone and Status
Clone: cloning a repository on our local machine 
	git clone <- some link ->
Status: displays the status of the code
	git status
There are 4 types of status:
1. Untracked: new files that git cannot track yet
2. Modified: changed
3. Staged: file is ready to be committed
4. unmodified: unchanged
Add and commit
Add: adds new or changed files in our working directory to the git staging area
	git add <- file name -> commit: it is the record of the change 
	git commit -m “some message”
Push command
push: upload local repository content to remote repository 
	git push origin main
Init command
Init command: used to create a new git repository 
	git init
	git remote add origin <- link -> 	git remote -v (to verify remote)
	git branch (to check branch)
	git branch -M main (to rename branch)
	git push origin main
Workflow
Local Git
1. Github repository
2. Clone
3. Changes
4. Add
5. Commit 
6. Push
Branch Command
	git branch (check branch)
	git branch -M man (rename branch)
	git checkout <- branch name -> (to navigate)
	git checkout -b <- new branch name  -> (to create new branch)
	git branch -d  <- branch name -> (to delete branch)
Merging code
Way 1:
	git diff <- branch name -> (to compare commit, branches, files and more)
	git merge <- branch name -> (to merge 2 branches)
Way 2:
	create a Pull Request (PR) -> PR lets you tell others about changes that we have pushed to a branch in a repository on GitHub.
Pull command 
	git pull origin main -> used to fetch and download content from a remote repository and immediately update the local repository to match that content.
Resolving merge conflict 
An event that takes place when Git is unable to automatically resolve differences in code between two commits.
Undoing changes
Case 1: staged changes
	git reset <- file name ->
	git reset
Case 2: committed changes (for one commit)
	git reset HEAD~1
Case 3: committed changes (for many commits)
	git commit <- commit hash ->
	git commit —hard <- commit hash ->
Fork
A fork is a new repository that shares codes and visibility settings with the original “upstream” repository 
Fork is a rough copy.
